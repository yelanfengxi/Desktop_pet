
# 【阶段一】测试指南

## 一、测试图片放置说明

### 1.1 图片格式要求
- 支持的图片格式：PNG（推荐，支持透明背景）、JPG、JPEG
- 图片尺寸：推荐 200x200 像素到 300x300 像素之间
- 文件命名：必须命名为 `pet.png`（或相应的格式后缀）

### 1.2 图片放置位置
将测试图片放置在以下目录中：
```
assets/default_models/pet.png
```

### 1.3 快速获取测试图片
如果您暂时没有合适的图片，可以：
1. 使用任意一张您喜欢的 PNG 图片
2. 或者从互联网上下载一张透明背景的角色图片
3. 将其重命名为 `pet.png` 并放置在指定目录

## 二、运行和测试步骤

### 2.1 启动开发环境
在项目根目录下打开终端（PowerShell 或 CMD），执行以下命令：
```bash
npm start
```

### 2.2 验证要点

#### 要点 1：窗口外观验证
- [ ] 窗口是否完全透明（无背景色）
- [ ] 窗口是否无边框
- [ ] 窗口是否始终置顶显示
- [ ] 窗口是否不在任务栏中显示

#### 要点 2：图片显示验证
- [ ] 图片是否正确加载并显示
- [ ] 图片是否居中显示在窗口中

#### 要点 3：鼠标穿透功能验证
- [ ] 当鼠标在透明区域（图片外）时，点击是否能穿透到下方的桌面内容
- [ ] 当鼠标悬停在图片上时，窗口是否能响应鼠标事件

#### 要点 4：拖拽功能验证
- [ ] 按住鼠标左键是否能拖动窗口
- [ ] 释放鼠标后窗口是否停止在新位置
- [ ] 拖拽过程是否流畅

## 三、常见问题排查

### 问题 1：图片无法加载
**解决方案**：
- 检查图片是否正确放置在 `assets/default_models/` 目录下
- 确认图片文件名是否为 `pet.png`
- 检查图片格式是否正确（推荐使用 PNG）

### 问题 2：窗口没有透明背景
**解决方案**：
- 确认系统是 Windows 11 或兼容的 Windows 版本
- 检查 main.js 中的 transparent 配置是否为 true

### 问题 3：鼠标穿透功能不工作
**解决方案**：
- 检查 main.js 中的 setIgnoreMouseEvents 调用
- 确认 IPC 通信是否正常工作
